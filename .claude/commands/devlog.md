현재 작업 내용을 분석해서 종족연구소 개발일지 초안을 작성합니다.

1. `git log --oneline -20` 과 `git diff HEAD~1` 으로 최근 커밋 및 변경사항을 확인한다.
2. 변경사항을 [신규 기능], [버그 수정], [개선 사항] 세 가지로 분류한다.
3. 아래 형식으로 개발일지 초안을 작성한다.
4. 사용자의 기존 개발일지 스타일을 유지한다.
5. 과장하지 말고 실제 작업한 내용만 작성한다.
6. 개발환경 구축, 조사만 한 내용은 제외한다.
7. 최종 결과만 출력한다. 분석 과정은 보여주지 않는다.

출력 형식:

개발일지 #[번호] [날짜]

[ 신규 기능 ]

▪기능 A 추가
▪기능 B 추가

[ 버그 수정 ]

▪버그 A 수정
▪버그 B 수정

[ 개선 사항 ]

▪UI 개선

- 해당 항목이 없으면 그 섹션은 생략한다.
- 번호는 git log에서 전체 커밋 수를 기준으로 추정한다.
- 날짜는 오늘 날짜(YYYY.MM.DD)를 사용한다.

사용자가 제공한 건의사항, 개선 요청, 기능 추가 아이디어를 분석합니다.

1. 사용자가 원하는 기능/개선사항을 한 줄로 요약한다.
2. 관련 화면/기능/DB/권한 구조를 현재 프로젝트 파일 기반으로 추정한다.
3. 확실하지 않은 내용은 "확인 필요"라고 표시한다.
4. 구현 방향은 최소/권장/확장 3단계로 나눠 제안한다.
5. 모르는 내용은 추측하지 말고 확인 필요라고 표시한다.
6. 말투는 간결하게, 개발자가 바로 작업할 수 있게 작성한다.
7. 불필요한 설명 없이 결과만 출력한다.

출력 형식:

[ 건의 내용 ]
▪ (사용자가 원하는 기능/개선사항을 한 줄로 요약)

[ 현재 구조 추정 ]
▪ (관련 화면/기능/DB/권한 구조 추정)
▪ (확실하지 않은 내용은 "확인 필요"로 표시)

[ 영향 범위 ]
▪ 관련 파일: (파일 경로)
▪ 관련 테이블/RPC/스토리지/권한 정책: (항목)
▪ 기존 기능 영향: (영향 설명)

[ 구현 방향 ]
1. 최소 구현안: (설명)
2. 권장 구현안: (설명)
3. 확장 가능안: (설명)

[ 확인할 것 ]
▪ (구현 전 확인해야 할 데이터 구조/파일/정책)
▪ (테스트해야 할 케이스)

[ 클로드에게 요청할 작업 ]
▪ (실제로 코드/SQL을 수정하기 위해 필요한 지시문)
